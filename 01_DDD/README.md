# DDD — Domain-Driven Design

Примеры и антипримеры Domain-Driven Design на C++ для курса «Архитектура программных систем».

## Описание

Набор примеров демонстрирует принципы DDD и типичные ошибки при проектировании доменной модели.

## Примеры

| Файл | Описание |
|------|----------|
| **example_0.cpp** | Антипример: анемичная модель, логика в сервисе, нарушение инкапсуляции |
| **example_1.cpp** | Улучшенная модель с инкапсуляцией |
| **example_2.cpp** | Value Objects |
| **example_3.cpp** | Агрегаты и границы |
| **example_4.cpp** | Доменные события |
| **example_5.cpp** | Репозитории и разделение чтения/записи |

## Ключевые концепции

- **Entity** — объект с идентичностью
- **Value Object** — неизменяемый объект без идентичности
- **Aggregate** — граница консистентности
- **Domain Event** — событие в домене
- **Repository** — абстракция доступа к данным
